# 🚀 한국 주식 자동분석 시스템

이 프로그램은 네이버 금융에서 한국 주식 정보를 자동으로 수집하고 분석하여 Google Sheets에 저장하는 완전 자동화 도구입니다.

## ✨ 주요 기능

- 📊 **전체 상장 주식 데이터 자동 수집** (코스피/코스닥)
- 🎯 **역발상 투자 후보 종목 자동 스크리닝**
- 📱 **Google Sheets 자동 업로드** (실시간 대시보드)
- 🔄 **완전 자동화** (데이터 수집 → 분석 → 업로드)
- 📈 **상위 종목 TOP 랭킹** (ROE, 시가총액 기준)

## 🛠️ 설치 방법

### 1. 저장소 클론
```bash
git clone https://github.com/belucci999/stock-program.git
cd stock-program
```

### 2. 가상환경 생성 및 활성화
```bash
python -m venv .venv
source .venv/bin/activate  # macOS/Linux
# 또는
.venv\Scripts\activate     # Windows
```

### 3. 패키지 설치
```bash
pip install -r requirements.txt
```
### 4. 프로그램 실행
python daily_auto_stock_analysis.py

### 4. Google Sheets API 설정

#### 4.1 Google Cloud Console 설정
1. [Google Cloud Console](https://console.cloud.google.com) 접속
2. 새 프로젝트 생성 또는 기존 프로젝트 선택
3. "API 및 서비스" → "라이브러리" 이동
4. "Google Sheets API" 검색 후 활성화
5. "Google Drive API"도 검색 후 활성화

#### 4.2 서비스 계정 생성
1. "API 및 서비스" → "사용자 인증 정보" 이동
2. "사용자 인증 정보 만들기" → "서비스 계정" 선택
3. 서비스 계정 이름 입력 후 생성
4. 생성된 서비스 계정 클릭 → "키" 탭 이동
5. "키 추가" → "새 키 만들기" → "JSON" 선택
6. 서비스 계정 키: `cursor/credentials/google-sa.json` (4전략 프로젝트와 공유)  
   - 별도 파일이 없으면 `.env`의 `CREDENTIALS_FILE=../credentials/google-sa.json` 사용

### 5. Google Sheets 문서 준비
1. [Google Sheets](https://sheets.google.com)에서 새 문서 생성
2. 문서 이름을 `주식분석결과_YYYYMMDD` 형식으로 설정
3. 서비스 계정 이메일을 문서 편집자로 공유 추가

### 휴장일·주말에 직전 거래일 기준 실행

스케줄러는 휴장일에 자동으로 건너뜁니다. **금요일 장 마감 데이터를 토·일·공휴일에 돌리려면:**

```bash
# 직전 거래일 탭(예: 2026-05-22)에 전체 전략 업로드

python daily_auto_stock_analysis.py --sheet-tab 2026-05-22
```

`--force` 미지정 시 `--sheet-tab`만 주면 `daily_auto_stock_analysis.py`에서도 동일하게 동작합니다.

### 구글 시트 (날짜별 탭)

`.env`에 `SPREADSHEET_ID`를 설정하고, 시트를 `google-sa.json`의 `client_email`에 편집 권한으로 공유합니다.

- 매일 실행 결과는 **`YYYY-MM-DD` 이름의 탭** 하나에 저장됩니다 (역발상·TOP500·20일선 돌파 등).
- 리바운드 전략은 같은 스프레드시트에 **`YYYY-MM-DD_거래량급감`** 등 별도 탭으로 추가됩니다.
- 같은 날 두 번 실행하면 **해당 날짜 탭을 비운 뒤 덮어씁니다** (나중 실행이 최종본).
- 탭 안에는 분석 요약, TOP500, 코스피/코스닥, 역발상 결과 등이 섹션별로 들어갑니다.

### 거래대금 컬럼

| 컬럼 | 계산 |
|------|------|
| 거래대금 | 당일 종가(현재가) × 당일 거래량 |
| 전일거래대금 | 전일 종가 × 전일 거래량 |
| 거래대금증감율 | (거래대금 − 전일거래대금) / 전일거래대금 × 100 (%) |

네이버에서 값을 못 가져온 경우 위 식으로 자동 채웁니다. `전일거래대금`, `거래대금증감율`은 `거래대금` 열 오른쪽에 위치합니다.

## 역발상 투자 후보 스크리닝 (`contrarian_stock_screener.py`)

전일 수집한 `full_stock_data_*.xlsx`를 읽어, **거래가 급격히 줄었지만 재무·규모는 양호한 종목**을 골라냅니다.  
PER·PBR은 **필터 조건에 사용하지 않습니다** (코드 주석: PER/PBR 조건 제외).

### 입력·파생 지표

| 지표 | 계산 |
|------|------|
| 가격변화율 | (현재가 − 전일종가) / 전일종가 × 100 (%) |
| 거래량변화율 | (거래량 − 전일거래량) / 전일거래량 × 100 (%) |

스크리닝 전에 `현재가`, `전일종가`, `거래량`, `전일거래량`, `ROE`, `시가총액`이 모두 있는 종목만 **유효 데이터**로 둡니다.

### 필터 조건 (모두 AND, 순서대로 좁혀 감)

| 단계 | 조건 | 의미 |
|------|------|------|
| 1 | 거래량변화율 ≤ **−85%** | 전일 대비 거래량이 85% 이상 감소 |
| 2 | 가격변화율 **< 0** | 당일 음봉(종가 하락) |
| 3 | 시가총액 ≥ **500** (억 원) | 규모 있는 종목만 |
| 4 | ROE **> 0** | 최근 기준 ROE 양수 |
| 5 | 전일거래량 ≥ **50,000**주 | 전일 유동성 확보 |
| 6 | 가격변화율 **−7% ~ −1%** | -7%보다 더 크게 빠진 종목 제외, -1% 미만 하락도 제외 |

**1~6을 모두 만족한 종목**이 `역발상투자후보`입니다.  
6단계를 통과하지 못하면 후보 0건이며, 이때는 **1~5만 만족**한 종목을 콘솔에 참고용으로만 출력합니다.

### 투자점수 (후보가 있을 때)

기본 점수 **100**에 아래 가점을 더해 **내림차순** 정렬합니다.

| 항목 | 가점 |
|------|------|
| 거래량변화율 ≤ −95% / −90% / −85% | +25 / +20 / +15 |
| ROE ≥ 20 / 15 / 10 / 5% | +20 / +15 / +10 / +5 |
| 시가총액 ≥ 1조 / 5천억 / 1천억 / 500억 (억 원) | +15 / +12 / +8 / +5 |
| 가격변화율 −4%~−2% / −5%~−1% | +10 / +5 |
| 전일거래량 ≥ 50만 / 20만 / 10만 / 5만주 | +10 / +7 / +5 / +3 |

### 결과 파일 (`contrarian_stocks_YYYYMMDD_HHMM.xlsx`)

| 시트 | 내용 |
|------|------|
| 역발상투자후보 | 최종 후보 + 투자점수 |
| 대형주 / 중형주 / 소형주 | 시총 1조 이상 / 1천억~1조 / 1천억 미만 분류 |
| 기본조건만족 | 1~5단계만 통과한 종목 |
| 필터링통계 | 단계별 남은 종목 수 |

구글 시트 업로드 시 후보가 1건 이상이면 `역발상 투자 후보` 섹션으로 함께 올라갑니다.

## 20일선 상향 돌파 스크리닝 (`ma20_breakout_screener.py`)

최신 `full_stock_data_*.xlsx`를 기준으로 동작합니다. **먼저 데이터 수집**이 필요합니다.

### 조건 (모두 충족)

| 조건 | 설명 |
|------|------|
| ROE > **5** | 엑셀 재무 데이터 기준 |
| 20일선 상향 돌파 | **전일** 종가 ≤ 전일 20일선 **그리고** **당일** 종가 > 당일 20일선 |
| 돌파일 **양봉** | 당일 종가 > 당일 시가 |
| 돌파일 **거래대금 ≥ 50억원** | 당일 종가 × 당일 거래량 |

20일선은 네이버 금융 일별 시세 종가로 계산합니다 (최근 20거래일 단순이동평균).

### 실행

```bash
# 전체 (ROE>5 종목마다 일봉 조회, 수십 분~1시간 이상 소요 가능)
python ma20_breakout_screener.py

# 테스트 (앞에서 30종목만)
python ma20_breakout_screener.py --limit 30
```

### 결과

- `ma20_breakout_YYYYMMDD_HHMM.xlsx`
- 돌파 종목의 기존 재무·시세 컬럼 + `MA20`, `돌파일거래대금`, `돌파일거래대금(억)`, `양봉` 등
- 구글 시트 당일 탭에 `--- 20일선 상향 돌파 ---` 섹션으로 자동 업로드 (`daily_auto` 또는 단독 실행 시)

## 리바운드 전략 (`daily_rebound_analysis.py` / `rebound_strategies_analyzer.py`)

네이버 금융 **일별 시세**로 종목별 과거 데이터를 조회한 뒤, 아래 3가지 패턴을 탐지합니다.  
이동평균은 **종가 단순이동평균(SMA)** 입니다.

| 전략 | 최소 일봉 | 구글 시트 탭 예시 |
|------|-----------|-------------------|
| 거래량 급감 | 3일 | `YYYY-MM-DD_거래량급감` |
| 45일선(세력선) | 60일 | `YYYY-MM-DD_45일선` |
| 360일선(장기 지지) | 360일 | `YYYY-MM-DD_360일선` |

`--sheet-tab`을 주면 접두 날짜가 바뀝니다 (예: `--sheet-tab 2026-05-22` → `2026-05-22_거래량급감`).  
신호가 있으면 `YYYY-MM-DD_전체` 탭에 3전략 통합 결과도 올라갑니다.

### 공통

- **데이터**: `full_stock_data_*.xlsx`가 24시간 이내에 있으면 재사용, 없으면 `quick_stock_check`와 동일 방식으로 수집
- **대상**: `is_regular_stock()` 통과 종목 (ETF·스팩 등 제외)
- **일봉 지표**: `price_change` = (종가 − 시가) / 시가 × 100 (%), `volume_change` = 전일 대비 거래량 변화율 (%)

### 일괄 실행

`python daily_rebound_analysis.py`(인자 없음) 및 `daily_auto` 5단계에서 **3전략 모두** 실행됩니다 (`analyze_stock`이 거래량 급감·45일선·360일선을 순서대로 검사).

```bash
python daily_rebound_analysis.py              # 3전략 일괄
python daily_rebound_analysis.py volume_drop  # 거래량 급감만
python daily_rebound_analysis.py ma45         # 45일선만
python daily_rebound_analysis.py ma360        # 360일선만
```

### 1. 거래량 급감 (`analyze_volume_drop`)

**최근 3거래일** 패턴 (그저께 → 어제 → 당일):

| 조건 | 기준 | 비고 |
|------|------|------|
| 거래량 폭등 | 어제 거래량 ≥ 그저께 × **5** (500%↑) | |
| 거래량 급감 | 당일 거래량 ≤ 어제 × **0.20** (20%↓) | 강한 신호: ≤ **0.12** (12%↓) |
| 음봉 | 당일 `price_change` ≤ **−1%** | 시가 대비 종가 하락 |
| 5일선 이격 (강한 신호만) | 종가가 5일 SMA 대비 **±10%** 이내 | 최근 5일 종가 평균 |

**신호 등급**

| 등급 | 조건 |
|------|------|
| **강함** | 폭등 + 급감(12%↓) + 음봉 + 5일선 10% 이내 |
| **보통** | 폭등 + 급감(20%↓) + 음봉 (5일선 조건 없음) |

**매수 참고**: 결과의 `buy_point` = 당일 **고가** (음봉 고가 돌파 시 매수 타이밍으로 표기).

### 2. 45일선 · 세력선 (`analyze_ma45`)

**최근 60거래일** 기준:

| 단계 | 조건 | 기준 |
|------|------|------|
| ① 급등 이력 | 60일 내 일봉 **전일 대비 +20%↑**인 날 존재 | 가장 최근 급등일 기준 |
| ② 하락 추세 | 급등 이후 구간 **고점 대비 현재가 −5% 이하** | |
| ③ 거래량 감소 | 급등 직후 5일 평균 거래량 대비, **최근 5일 평균 < 70%** | 약 30% 이상 감소 |
| ④ 45일선 첫 터치 | 현재 종가가 **MA45 대비 −2% ~ +2%** | 급등 이후~전일까지 동일 구간 터치 이력 없음 |

MA45 = 종가 45일 SMA. ①~④를 모두 만족할 때 신호.

### 3. 360일선 · 장기 지지 (`analyze_ma360`)

**최소 380거래일** 일봉, **최근 60일** 패턴 분석:

| 단계 | 조건 | 기준 |
|------|------|------|
| ① MA360 우상향 | 최근 20일 MA360 기울기 **> 0** | |
| ② 강한 이탈 | 60일 내 어느 날이든 종가가 MA360 대비 **−10% 이하** | 이탈일 기록 |
| ③ 회복 시도 | 이탈 이후 **MA360 대비 −3% 이내**까지 가격 회복한 날 존재 | |
| ④ 현재 지지 | 현재 종가가 MA360 대비 **−5% ~ +3%** | |
| 신호 강도 | 최근 5일 평균 거래량 > 그 이전 15일 평균 × **1.2** | 충족 시 **강함**, 아니면 **보통** |

①~④를 모두 만족할 때 신호 (④는 필수, 거래량은 강도만 구분).
